Automation Workflow for Boliglejedata – Kommuner Tab

This section describes the precise automation flow for extracting residential lease data from the “Boliglejedata” module when logged in on ReData’s Boliglejedata Page.

Step 0: Open “Boliglejedata” from the Sidebar Menu

  • After login, locate the vertical navigation sidebar on the left.
  • Click on the tab labeled “Boliglejedata”. This opens the lease data module.
  • Wait until the page has fully loaded (URL should include /boliglejedata).

Step 1: Navigate to the "Kommuner" Tab

  • Locate the section or filter labeled “Kommuner” on the left-hand panel.
  • Click it to expand the list of municipalities.
  • Use the input field to search and select a municipality (e.g., Ballerup).
  • Wait for the map and right-hand analytics panel to reload with new data.

Step 2: Extract Metrics from Right Panel

Once the municipality is selected, extract the following data from the DOM elements on the right side:

  • Antal lejeopslag: Use query selector: 
    document.querySelectorAll('.charTableDataFont .text-left')[0].innerText
  • Gns. leje pr. m²: 
    document.querySelectorAll('.charTableDataFont .text-left')[1].innerText
  • Gns. antal værelser: 
    document.querySelectorAll('.charTableDataFont .text-left')[2].innerText
  • Gns. liggetid (dage): 
    document.querySelectorAll('.charTableDataFont .text-left')[3].innerText

These values are found inside an HTML table with classes like charTableDataFont or housingTableTotalDisplay. Use a headless browser automation tool (e.g., Selenium or Puppeteer) to parse and extract the data programmatically.

Step 3: (Optional) Extract Scatterplot or Trend Data

The scatterplot titled "Udvikling over lejepriser" can be accessed by:

  • Targeting the <canvas> element (may require pixel decoding)
  • Or accessing JavaScript-injected data sources (e.g., window.__REACT_QUERY_CACHE__ or network payloads)

Additional Notes

  • Ensure page elements are fully rendered before querying (observe network idle or use waitForSelector).
  • After selecting a new municipality, DOM structure will re-render — re-evaluate selectors.
  • Log and cache extracted values with a timestamp + municipality name.

Available Modules

Each module corresponds to a tab or navigation item on the platform UI.

OneDoor

Aggregates data from >20 sources (e.g., Tinglysning, BBR, CVR). Provides:

  • Property documents: Tingbog, lokalplan, servitutter
  • Company insights: Regnskaber, ejerstruktur
  • Person-level relationships
  • Live external links: SKAT, Boligejer, Byggesagsarkiv

Boligdata

Insights into private housing market (e.g. villa, rækkehus, ejerlejlighed).

  • Trends: m²-priser, liggetid, afslag
  • Search by area and time (last 4 years)
  • Downloadable transaction-level data

Analyse

Market trends and transaction volume since 2008.

  • Biggest buyers/sellers
  • Segment and region-based filters
  • Export charts to PNG or Excel

Transaktionsdata

In-depth database of commercial and investment property transactions.

  • Includes lease, yield, OpEx, risk, usage type
  • Filter: area, m², return, build year, type
  • Used for: Underwriting, Due Diligence

Markedsindsigt

Detailed overview of the Danish office market.

  • Split into 19 subregions with descriptions
  • Pie charts of building use types
  • Tabular data: leases, supply, avg. gross rent/m²

Note: Accessible to both logged-in and anonymous users.

Boliglejedata

Lease data for residential units across Denmark.

  • Metrics: Rent levels, trends, area comparisons
  • Filter by: area, size, build year, features, price
  • Provides comparables for rental pricing and investment cases

Automation Tips

  • Ensure browser-based automation tools support JavaScript rendering (use headless Chromium with Puppeteer or Selenium).
  • Wait for DOM elements to load on each tab (e.g. `document.querySelector(".property-list")`).
  • Use `localStorage` or `cookies` to manage login session if scraping multiple modules.
  • Export features may return CSV, XLSX, or image formats.
